### 🧠 System Architecture & Core Concepts
This project acts as an autonomous digital auditor. It parses raw text data, categorizes risk levels using `Keyword` extraction, and generates structured reports.

* **Object-Oriented Programming (OOP):** The entire logic is encapsulated within the `AuditAnalyzer` `Class`, ensuring reusability, modularity, and clean `State Management`.
* **Dynamic Data Parsing:** Moves beyond unreliable `Directory` structures by analyzing the actual file content to determine compliance (e.g., `AI_Compliance`, `High_Risk`).
* **Robust Exception Handling:** Production-ready `try-except` blocks ensure the system gracefully handles missing files or read errors without crashing.
* **Persistent Auditing Trails:** Utilizes the Python `logging` module to generate a real-time `audit.log` file, recording every processed file and system event.
* **Standardized Data Export:** Compiles findings in memory using nested `Data Structures` (`Dictionary` & `List`) and exports them as a universally readable `JSON` format.

### 📂 File Structure
* `main.py`: The core application containing the `AuditAnalyzer` `Class` and `Runtime` logic.
* `audit_data/`: A directory containing raw `.txt` files representing raw audit evidence.
* `audit.log`: Real-time system execution logs (Excluded via `.gitignore`).
* `audit_report.json`: The final structured findings generated by the system (Excluded via `.gitignore`).
